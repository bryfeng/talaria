## Components

### server.py — Flask API Server

The core of Talaria. Serves both the REST API and the static web UI.

| Property | Value |
|----------|-------|
| Port | 8400 (TALARIA_PORT) |
| Data store | board.json |
| Optional docs | cards/*.md |
| Templates | Jinja2 via Flask |
| Static files | static/ |

**Key Routes:**

| Method | Path | Description |
|--------|------|-------------|
| GET | /api/board | Full board state (columns + cards + activity) |
| POST | /api/card | Create card |
| GET | /api/card/:id | Get single card |
| PATCH | /api/card/:id | Update card (column, priority, labels, etc.) |
| DELETE | /api/card/:id | Delete card |
| POST | /api/card/:id/note | Add a status note |
| GET | /api/activity | Recent activity log |
| GET | / | Web UI |

### agent_watcher.py — Agent Orchestrator

Long-running process that watches the board and dispatches autonomous agents.

**Lifecycle for a card in `in_progress`:**

```
1. agent_watcher polls GET /api/board
2. Finds cards in "in_progress" column
3. For each card without an active agent:
   a. Load card context (talaria context <id> | jq)
   b. Create git worktree + branch
   c. Spawn subagent (Hermes Agent) with card context
   d. Log worktree path + branch to card.status_notes
4. When card moves to "done":
   a. agent_watcher detects column change
   b. Cleans up worktree
   c. Optionally: LLM auto-summary, cost log
```

**Environment Variables:**

| Variable | Default | Purpose |
|----------|---------|---------|
| TALARIA_PORT | 8400 | Server port |
| TALARIA_HOME | ~/.talaria/talaria | Config path |
| TALARIA_WORK_DIR | ~ | Agent working directory |
| TALARIA_MAX_CONCURRENT | 2 | Max simultaneous agents |
| HERMES_AGENT_PATH | ~/.hermes/hermes-agent/run_agent.py | Agent binary |
| HERMES_VENV_PATH | ~/.hermes/hermes-agent/.venv/bin/python | Python interpreter |

### talaria_cli.py — Terminal Interface

For terminal-first workflows and agent scripting.

```bash
ln -s $(pwd)/talaria_cli.py /usr/local/bin/talaria

talaria list                    # All cards grouped by column (JSON)
talaria create <title>          # New card in backlog
talaria move <card-id> <col>   # Move card (triggers agent_watcher)
talaria log <card-id>           # Activity log + status notes
talaria context <card-id>       # Full card data (for agent prompts)
```

All output is JSON for easy piping:

```bash
# Get card description for agent
talaria context 9f875537 | jq '.description'

# List all review cards
talaria list | jq '.[] | select(.column == "review")'
```

## Trigger System

Columns can have a `trigger` that fires automation when a card enters that column:

| Trigger | Handler | Description |
|---------|---------|-------------|
| `null` | — | No action |
| `agent_spawn` | agent_watcher.py | Dispatch AI subagent |
| `notify` | _notify_telegram() | Send Telegram message |
| `webhook` | _call_webhook() | POST to configured URL |

Extending triggers in `server.py`:

```python
if trigger == "agent_spawn":
    _queue_agent(card)
elif trigger == "notify":
    _notify_telegram(card)
elif trigger == "webhook":
    _call_webhook(card, column_config)
```
