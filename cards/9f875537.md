## Problem
agent_watcher.py only watches a queue file and spawns hermes. It needs to be a proper pipeline runner that reads column config, drafts context, and dispatches the right worker.

## Solution
Refactor agent_watcher.py into a pipeline runner. Key changes:

1. **Column trigger detection**
   - Instead of just watching agent_queue.json, poll /api/board for cards in trigger columns
   - Track which cards are "in flight" — dont re-dispatch already-working cards

2. **Column config** (read from kanban.json columns[])
   - worker: hermes | claude-code | codex
   - context_files: [file paths relative to TALARIA_HOME/agents/]
   - instructions: optional templated string

3. **Context drafting**
   - Read TALARIA_HOME/talaria.md (always first)
   - Read each file in context_files[]
   - Concatenate with card spec at the end
   - Write to temp file for the worker

4. **Worker dispatch**
   - hermes: spawn subagent with --goal pointing to temp context file
   - claude-code: invoke CLI with context file
   - codex: invoke CLI with context file
   - Track PID for lifecycle management

5. **Lifecycle**
   - On spawn: update card with agent_session_id=PID
   - Poll PIDs every 30s
   - On exit: log cost, add completion note, advance card

6. **Cost tracking**
   - Store {tokens, cost_usd, ts} on card.cost_log
   - Parse from agent output or estimate from runtime

## API changes
None — uses existing /api/card and /api/board endpoints.

## Files changed
- agent_watcher.py: full rewrite
- server.py: add worktree_path, branch_name, cost_log fields to card schema
